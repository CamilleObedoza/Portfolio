SQL Project

A set of SQL queries built to answer the same handful of business questions that kept getting asked, so people would stop rebuilding the same query slightly differently every time.

SQL Relational Databases Documentation

Overview

The same business questions — order counts by region, top categories by revenue, month-over-month growth — kept coming up, and each time someone rebuilt the query from scratch, often slightly differently. I wrote and tested a reusable set of queries, checked the results against known totals to confirm accuracy, and documented the logic behind each one.

Testing against known totals actually caught a real problem: a few existing ad-hoc queries were silently double-counting orders because of an unhandled join, which had been quietly inflating past reports.
